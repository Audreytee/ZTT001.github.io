基于PhysX的Cloth学习笔记

用户指南

角色服装的逼真运动极大地改善了玩家体验。 PhysX 3布料功能是一种完整的高性能解决方案，可用于模拟角色服装。 它提供高精度和稳定性的局部空间模拟，减少拉伸的新技术，与各种形状的碰撞，以及颗粒自碰撞和相互碰撞，以避免布料穿透自身或其他布料实例。 可以将仿真卸载到支持CUDA的GPU，以获得更好的性能，或者以高于CPU能够处理的分辨率运行资产。

PhysX 3布料是PhysX 2变形的重写，专为模拟字符布料而定制。 软体，撕裂和双向交互已被删除，而布料模拟的行为和性能已得到改善。

simulation algorithm

对于一个PhysX模拟框架，布料解算器运行多次迭代。 迭代次数由求解器频率参数和模拟帧时间确定。 每次迭代都会集成粒子位置，解决约束，并执行角色和自碰撞。 在场景中的所有布料实例逐步前进后，每帧执行一次布料相互碰撞。 从用户指定的每帧值开始，每次迭代内插局部帧，运动约束和碰撞形状。

求解器频率指定为每秒迭代次数。 求解器频率值240对应于每帧以每秒60帧的4次迭代。 通常，如果使用更高的求解器频率值，模拟将变得更准确。 然而，仿真时间与求解器频率大致呈线性增长。 通常，该值介于120和300之间。
使用模拟频率和模拟时间步长导出每帧的迭代次数。 PhysX尝试通过在位置积分期间和应用阻尼参数（如约束刚度）时考虑时间步长的变化来仔细处理可变时间步长。 虽然这减少了由于时变步长变化引起的可能的抖动伪像，但通常不建议使用可变时间步长。

Particle Integration

布料迭代的第一步是根据其当前位置，速度和外部加速度预测新的粒子位置。 当粒子状态由当前位置和最后一次迭代之前的位置组成时，局部空间中的粒子速度可以通过将位置delta除以前一次迭代的delta时间来计算。

Constraints

在粒子位置被整合之后，解决一组不同的约束以模拟拉伸，剪切和弯曲力，以及将粒子运动限制在特定区域内。

Distance Constraints

Tether Constraints

距离约束每次迭代仅解决一次而不完全收敛。这种近似的最明显的神器是布料变得有弹性。增加求解器频率会减少拉伸，但会导致模拟时间增加。

PhysX 3.3引入了系绳约束作为解决方案，以避免在重力或快速运动下的伸展性。系绳约束通过限制粒子可以远离其锚定粒子的距离来防止拉伸。此约束为求解器添加了非常少量的计算，因此它比增加迭代次数更有效。

当一些PxClothMeshDesc :: invMasses值设置为零时，炊具会自动生成系绳约束，告诉炊具相应的粒子是非模拟锚粒子，其位置仅由用户提供。在创建结构后更改反向质量不会影响将哪些锚定粒子用于系绳约束。

Motion Constraints

可以将点完全约束到具有零反质量的用户指定位置。 但是，有时需要将一个点限制在动画（用户指定）位置周围的小区域内。 这允许通过模拟生成小细节，同时抑制与期望位置的任何过度偏差。

运动约束锁定每个粒子在球体内的运动。 例如，动画系统可以绘制布料的整体运动，而细节比例细节由布料模拟处理。

Separation Constraints

分离约束与运动约束完全相反，迫使粒子停留在球体之外。 当粒子运动受到运动约束（例如臂周围的套筒）的适度约束时，可以使用分离约束来比单独使用胶囊更准确地表示角色的碰撞形状。 例如，通过将半径设置为从球体中心到角色表面的距离，可以将分离约束放置在角色内部。

请参阅PxClothParticleSeperationConstraint和PxCloth :: setSeparationConstraints（）。

Collision Detection

每个布料对象都支持与球体，胶囊，平面，凸面（平面组）和三角形的碰撞。默认情况下，这些形状都与主PhysX刚体场景分开处理，但是可以使用PxClothFlag :: eSCENE_COLLISION标志启用与其他PxScene actor的碰撞。

在模拟场景之前，在下一帧的本地坐标中指定碰撞形状。使用从帧的开始和结束处的值插值的形状位置，执行独立且完整的碰撞阶段作为每个求解器迭代的一部分。球体和胶囊碰撞支持连续碰撞检测，并使用加速结构在碰撞管道的早期剔除远处的粒子。因此，球体和胶囊是模拟字符形状的首选，而凸起和三角形应该只是谨慎使用。

球体定义为中心和半径。请注意，特别允许半径在帧与帧之间变化。每块布的球体总数限制在32个。

胶囊由球体阵列中的一对指数限定，并且每个球体可具有不同的半径，从而形成锥形胶囊。球体可以在多个胶囊之间共享，这可以用于模拟角色（由胶囊组成的上腿和下腿可以在膝盖处共享球体）。分享球体也使模拟更有效和更强大，因此非常鼓励。


平面由它们的法线和距原点确定。 它们不会被认为是碰撞，除非它们以凸形状引用。 使用掩码的凸面参考平面，其中每个位对应于平面阵列中的条目。 每块布限制为32架。

三角形碰撞器被定义为逆时针缠绕顺序的顶点三元组。 三角形应在布料附近形成一个封闭的补丁，以实现一致的碰撞处理（每个粒子碰撞其最接近的三角形，扩展到无限平面）。

平面和三角形的顺序应保持不变（除了通过PxCloth :: removeCollisionPlane / Triangle（）方法移除它们），因为它们的位置在模拟帧之间进行插值。

Continuous Collision Detection

除了在每次迭代结束时解析形状内的粒子的离散碰撞，还支持连续碰撞检测，并且可以通过调用来启用：

连续碰撞的计算成本比离散碰撞大2倍，但有必要检测快速移动物体之间的碰撞。 连续碰撞分析颗粒和胶囊的轨迹以确定何时发生接触。 在第一次接触之后，粒子随着形状移动直到迭代结束。

Virtual Particle Collision

虚拟粒子提供了一种改善布料碰撞的方法，而不会增加布料的分辨率 它们被称为“虚拟”粒子，因为它们仅在碰撞处理阶段存在，并且没有像常规粒子那样明确地存储它们的位置，速度或质量，它们可以被认为是在碰撞表面上提供额外的样本。

在碰撞处理期间，使用重心插值从三个正常粒子创建每个虚拟粒子。 然后像常规粒子一样测试离散碰撞，并使用反向插值将碰撞脉冲重新分配回原始粒子。

“添加虚拟粒子”一节介绍了使用此功能的必要步骤。

自碰撞距离定义了每个粒子周围球体的直径，并且求解器确保这些球体在模拟过程中不会重叠。 自碰撞刚度定义了分离脉冲应该有多强。

自碰撞距离应小于其余配置中两个粒子之间的最小距离。 如果距离较大，则自碰撞可能违反某些距离约束并导致抖动。
